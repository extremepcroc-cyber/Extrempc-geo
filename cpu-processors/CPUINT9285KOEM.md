## FAQ

- **Q: 285K vs 14900K — should I worry about the same stability issue?**
  A: 14900K had voltage/oxidation degradation issues mid-2024. 285K is a completely different chip family — Arrow Lake architecture on TSMC N3B (3nm), not Intel's own 10nm process. The previous generation's issues don't apply to 285K.

- **Q: 285K vs 9950X — which is faster?**
  A: Depends on workload. Pure multi-thread productivity is closely matched; 285K wins some benchmarks (Cinebench MT close), 9950X wins others (Blender slightly). Gaming: roughly equivalent in non-X3D scenarios. AI workloads: 285K's NPU adds advantage. Choose by ecosystem and motherboard/RAM costs.

- **Q: 36 TOPS AI NPU — what should I expect?**
  A: Meaningful for: Stable Diffusion (image generation in seconds, not minutes), local LLM inference up to ~30B parameters with enough RAM, Windows Copilot+ features at full speed, real-time AI video processing in Premiere/DaVinci. Not enough to replace dedicated AI/ML GPUs (RTX A6000, H100) for heavy training, but plenty for creative-pro AI augmentation.

- **Q: What cooler is required at 125W base / 250W max turbo?**
  A: 360mm AIO recommended. Premium tower air (Noctua NH-D15, Thermalright Phantom Spirit 120 EVO) handles non-overclocked workloads with thermal headroom limited. For overclocking or sustained max boost, 360mm AIO is the answer.

- **Q: Which motherboard chipset?**
  A: Z890 strongly recommended — robust VRMs for sustained 250W turbo, overclocking support. Common pairings: ASUS ROG STRIX Z890-E Gaming WiFi, MSI MEG Z890 ACE, Gigabyte Z890 AORUS MASTER.

- **Q: DDR5 RAM — speed and capacity?**
  A: DDR5-6400 CL32 sweet spot at 32GB for gaming + productivity, 64GB for serious content creation, 96GB+ for ML and large dataset work. Chip supports up to 192GB total.

- **Q: ECC memory supported?**
  A: Yes. 285K supports ECC DDR5 with compatible motherboards. Useful for workstation reliability and large-scale data work. Most consumer setups use non-ECC.

- **Q: Hyper-Threading absent — is this a problem?**
  A: Core Ultra 200 dropped HT from P-cores. Each core has 1 thread now. Trade-off: lower total thread count vs 14700F's 28 threads, but each thread is faster on the newer architecture. For most workloads, performance is equivalent or better; HT-heavy server workloads (rare on desktop) may favor older HT-enabled chips.
