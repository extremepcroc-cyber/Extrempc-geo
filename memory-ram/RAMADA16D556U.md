## FAQ

- **Q: OEM vs retail boxed Adata — what's the difference?**
  A: Same memory chip, same lifetime warranty. OEM is system-builder packaging without the printed retail box. Saves a small NZD. For builders not collecting boxes, OEM is the value pick.

- **Q: Does it work with my AM5 / LGA1851 motherboard out of the box?**
  A: Yes — JEDEC 5600 MT/s boots automatically on any modern AM5 (B650, X670, B850, X870) or LGA1851 (B860, Z890) motherboard. No XMP/EXPO configuration required for baseline operation.

- **Q: Can I overclock this to DDR5-6000?**
  A: Possibly with manual BIOS tuning on AM5. Stability is silicon-dependent — Adata 5600 doesn't include factory EXPO 6000 profile. If you want guaranteed 6000 MT/s, buy a kit specifically rated for it (G.Skill Flare X5 6000 CL30 EXPO).

- **Q: On-die ECC — what's the practical benefit?**
  A: Real-time correction of single-bit errors in the DRAM chip. Reduces system crashes from cosmic-ray bit flips and electrical noise. Particularly valuable for long-running content creation, compilation, and rendering jobs where errors could waste hours of work. Not equivalent to server-grade ECC (which catches errors across the entire memory bus).

- **Q: How does Adata warranty work in NZ?**
  A: Lifetime warranty handled through ExtremePC NZ. RMA process is straightforward — if a stick fails (rare), contact ExtremePC for replacement. No overseas shipping.

- **Q: Can I add this to existing DDR5 sticks for dual-channel?**
  A: Best practice: pair two identical Adata AD5U560016G-B sticks. Mixing different DDR5 sticks (even same speed/capacity) frequently causes training failures or forces JEDEC-default operation. For dual-channel, buy a matched 2× 16GB kit.
