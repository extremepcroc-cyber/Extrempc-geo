## FAQ

- **Q: Will this work in my consumer desktop motherboard (Ryzen 5000, Intel 12th-14th Gen)?**
  A: **No.** ECC Registered RAM requires specific server / workstation motherboard chipsets: AMD Threadripper PRO (TRX50, WRX80, WRX90), Intel Xeon W (W680, W790), Intel server (C-series), AMD EPYC (server). Standard consumer chipsets (B550, X570, B650, B850, Z690, Z790, Z890) do **not** support ECC Registered RAM — system will not boot or will run with errors.

- **Q: What's the difference between ECC Registered (RDIMM) and ECC Unbuffered (UDIMM)?**
  A: RDIMM (this product) has a register/buffer on the module that regenerates signals — enables high-DIMM-count populations (8+ DIMMs per board) at full speed. Required by most servers and Threadripper PRO. UDIMM is unbuffered ECC — works in some consumer Ryzen + ASRock Rack motherboards but limited to 2 DIMMs per channel max. Confirm your motherboard requires RDIMM specifically before buying.

- **Q: Do I really need ECC for my workstation?**
  A: For scientific computing, ML training, financial modeling, long-running content rendering, or any workload where silent corruption could invalidate results — yes, ECC is essential. For typical office / gaming use — no, consumer non-ECC RAM is sufficient.

- **Q: Bit flip rate — how often does this actually happen?**
  A: Studies estimate cosmic-ray bit flips at ~1 per GB per month on consumer DDR4. For a 256GB workstation running 24/7, that's potentially several bit flips per day. Most cause no visible issue, but rare ones can crash systems or silently corrupt data. ECC catches and corrects them.

- **Q: Will it run at 3200 MT/s in my system?**
  A: Yes on supported server/workstation motherboards at JEDEC 3200. With 8+ DIMMs populated, speed may auto-drop to 2933 or 2666 due to memory controller load (this is normal RDIMM behavior).
